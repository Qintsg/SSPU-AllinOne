# 学工报表第二课堂学分查询规则探索

本文记录 issue #115 与 issue #182 对学工报表系统的只读探索结果，后续维护第二课堂学分查询时以本文为约束。

## 入口与认证

- OA 学工报表入口为 `https://oa.sspu.edu.cn/interface/Entrance.jsp?id=xgreport`。
- 未认证访问 OA 入口时，会先跳转到 `https://oa.sspu.edu.cn/sso/login.jsp`，再跳转到 `https://id.sspu.edu.cn/cas/login`。
- 已认证访问 OA 入口时，OA 门户页提供 `https://xgbb.sspu.edu.cn/sharedc/sso/fore-login.do`，需先打开该 SSO 链接换取 `xgbb` 业务会话。
- `xgbb` SSO 链路可能返回 `http://id.sspu.edu.cn/cas/login` 跳转，客户端统一改用 HTTPS CAS 地址，避免卡在不可用的 HTTP 入口。
- 学工报表业务首页为 `https://xgbb.sspu.edu.cn/sharedc/core/home/index.do`。
- 未认证访问业务首页时返回 `200 OK`，页面脚本跳转到 `https://xgbb.sspu.edu.cn/sharedc/core/login/index.do?protocol=https`。
- 本地登录页包含用户名、密码与验证码字段，当前实现不提交该表单；应用仅通过 OA/CAS 入口复用已保存的 OA 登录态。
- 每次查询前必须先执行校园网 / VPN 前置检测，不可达时不打开 OA 入口、不刷新 OA 会话。
- 若 OA 入口、学工报表 SSO、业务首页或第二课堂明细页返回 CAS / 本地登录页，且本地已保存 OA 密码，应用只刷新一次 OA/CAS 会话并从 OA 入口重新进入学工报表；重试后仍要求登录时返回登录态不可用。

## 页面与接口线索

- 登录后先打开 OA `xgreport` 入口，再读取学工报表首页。
- 第二课堂学分入口在登录后页面中以“第二课堂学分查询”“第二学堂学分查询”或“学分 + 查询”文字出现，可能位于 `a`、`button`、`li`、`span` 或 `div` 元素中。
- 实现只从明确入口的 `href`、`data-url`、`data-href`、`url` 或 `onclick` 中解析 `.do` 跳转地址。
- 当前真实菜单通过 `toMainUrl('dc/studentxfform/index.do', ...)` 指向 `https://xgbb.sspu.edu.cn/sharedc/dc/studentxfform/index.do`，该页面可用 GET 只读读取学分表。
- 第二课堂主页面优先按规则矩阵解析，表头字段为“类别 / 项目 / 等级 / 参与情况 / 积分 / 已获分数（或已获积分）/ 必修积分 / 通过情况”。
- 规则矩阵存在大量 `rowspan` / `colspan` 合并单元格，解析前必须展开合并单元格，避免类别、项目、积分、已获分数、必修积分错位。
- “总计 / 合计”行单独解析为总积分、总已获分数、总必修积分和总体通过情况，不混入明细记录。
- “已获分数 / 已获积分”单元格中的 `href`、`data-url`、`data-href`、`url`、`onclick`、`data-options`、子元素属性或单元格内脚本片段可能包含详情入口；只允许 GET 读取明确解析到的详情页或解析已返回的 HTML。
- 已获分数详情表字段为“名称 / 类别 / 项目 / 等级 / 参与情况(h) / 获得积分 / 签到时间”，本地只保存名称、类别、项目、等级、参与情况和获得积分，忽略签到时间等非必要字段。
- 旧版第二课堂页面可继续按表格 fallback 解析明细，优先识别“类别 / 类型 / 模块”、“项目名称 / 活动名称 / 课程名称”、“学期 / 学年”、“时间 / 日期 / 认定时间”、“状态 / 审核 / 认定”和“学分 / 分值 / 得分 / 分数 / 认定分 / 获得分”等列。
- 真实明细表可能不提供独立学期分组行，单项记录中的 `2024-2025-1` 等学期单元格应归入该条记录的学期字段，不得误作项目名称或学分。
- 状态字段只保留明确状态值，例如“通过 / 合格 / 已通过 / 完成 / Pass / 不通过 / 不合格 / 未通过 / 失败 / Fail / 已认定 / 审核通过”等；“活动”等课程类型或类别不得写入状态字段。
- 解析成功后保存规则矩阵、总计和已获分数详情；旧版 fallback 逐项保留项目名称、类别、得分、学年学期、时间、状态和原始单元格。连续明细行的类别单元格为空时沿用上一条有效类别，不将单项得分相加作为官方总学分。
- 第二课堂缓存属于鉴权业务数据，必须写入系统安全存储。旧版普通应用状态中的第二课堂明文缓存不迁移，读取或写入安全缓存时直接清除同集合旧普通缓存。
- 缓存 payload 不得包含账号明文、Cookie、CAS Ticket、未脱敏 URI query 或其它登录材料。

## 刷新与展示策略

- 教务中心页“第二课堂学分”卡片默认不自动读取，避免进入页面即访问需要校园网 / VPN 与 OA 登录的受限服务。
- 可点击卡片右上角刷新图标手动读取，卡片右下角显示上次刷新时间。
- 可在设置页“自动刷新设置”中开启“第二课堂学分自动刷新”并选择刷新间隔；开启后进入教务中心会主动读取一次。
- 查询成功后卡片展示总已获分数、总必修积分、总体通过情况、详情记录数和上次刷新时间。
- 详情页展示顶部总计概览，下方分别展示规则矩阵和“已获分数详情”表；窄屏下表格需横向滚动或堆叠展示，避免溢出。
- 查询失败时卡片展示凭据缺失、校园网不可达、OA 登录态不可用、入口缺失或页面结构异常等明确状态。
- 主矩阵解析成功但部分详情页读取或解析失败时，当前结果可展示明确 warning，但不得覆盖上一份完整安全缓存。

## 只读边界

- 当前能力只允许访问 OA/CAS 登录链路、学工报表首页、明确解析到的“第二课堂学分查询”只读页面和“已获分数 / 已获积分”只读详情页。
- 不执行新增、修改、删除、提交、申报、审核、撤销、导出、发送消息等任何写入或状态变更操作。
- 不提交 `xgbb` 本地登录页验证码表单，也不尝试绕过验证码、MFA 或其它交互式安全校验。
- UI、日志和文档不得展示 OA 密码、Cookie、CAS Ticket 或其它可直接复用的身份值。

## 未确认风险

- 学工报表系统可能按角色、年级或权限展示不同菜单；若入口缺失，应用应返回“未找到第二课堂学分查询入口”。
- 若页面结构变化导致无法解析，应用应返回“页面结构异常”，不得伪造学分、类别或空成功结果。
