# 学校邮箱协议接入规则

## 账号与凭据

- 学校邮箱账号固定由学工号派生为 `学工号@sspu.edu.cn`。
- 邮箱密码独立于 OA 密码，统一通过系统安全存储保存。
- 不得在测试、日志、PR、Release notes 或普通缓存中写入真实邮箱密码、真实邮件正文、真实收件人或真实个人邮件内容。

## 收信边界

- IMAP / POP 仅用于读取最近邮件快照。
- IMAP 使用只读语义读取正文快照，POP 只检索最近邮件。
- 不提供删除、移动、标记已读、修改文件夹、批量处理、自动回复或转发入口。
- 成功读取的邮件摘要缓存按鉴权业务缓存策略保存；账号明文不写入普通 `app_state.json`。

## SMTP 发信边界

- SMTP 可用于登录校验，也可在用户主动点击“发送邮件”时发送普通文本邮件。
- 发信表单当前支持 To、Cc、Bcc、主题和纯文本正文。
- 暂不支持附件、富文本编辑、草稿、定时发送、自动发送、后台重试或静默发送。
- 发信请求提交后，以学校邮箱服务端状态为准；应用不尝试撤回或自动重发。
- 发信正文、主题、收件人、抄送和密送只用于本次 SMTP 请求，不写入普通本地缓存，不写入日志。

## 数据与隐私

- 用户主动发信时，收件人、抄送、密送、主题和正文会提交给学校邮箱 SMTP 服务端处理。
- 外部邮箱服务的日志、投递状态、风控策略和服务端数据保留规则由服务提供方负责。
- 页面反馈只展示脱敏状态文案，不回显完整收件人列表或正文。

## 测试策略

- 单元测试必须使用 fake gateway，不连接真实邮箱服务。
- 测试用地址和正文必须使用虚构数据，不得写入用户真实账号或真实邮件内容。
- 发信测试需覆盖：未保存凭据、输入非法不触网、SMTP 成功、SMTP 被拒绝、网络失败、普通缓存不出现发件明文。
